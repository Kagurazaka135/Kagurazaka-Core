# memory.py — 轻量级记忆系统

**让 AI 记得"刚才说了什么"。** 不是 RAG，就是最朴素的上下文追踪 + 搜索缓存。

## 三层记忆 + 缓存

```
┌─ 工作记忆 (deque, 最大 7 条)
│  当前会话的关键信息, FIFO
│
├─ 会话历史 (deque, 最大 10 条)
│  最近 10 轮对话的完整记录
│
├─ 搜索缓存 (deque, 最大 20 条)
│  搜索关键词 → 结果，30min TTL，关键词模糊匹配
│
└─ 长期记忆 (磁盘 JSON)
   按 chat/code/logic 分类存储，每类上限 15~50 条
```

## 核心方法

### enhance_input(user_input) → str
检测上下文引用（正则匹配"刚才/刚刚/这个/那个/继续"等指代词），或者检测话题关键词重叠。
自动从历史里捞出上文拼接。

### add_to_history(user_input, ai_output, task_type)
每轮对话结束时调，保存进历史。超过 200 字的输出或非聊天类型的任务，额外保存到长期记忆。

### add_search_cache / find_in_cache
搜索前先查缓存：精确匹配关键词 → 模糊匹配 → 缓存过期跳过。
缓存超 30min 过期，但会先复用旧结果再异步更新。

### update_task_context(task_type, step_info)
多步骤任务追踪。比如用户连续问代码问题，系统知道"上一个任务是代码，已完成 3 步"。

## 中文关键词提取

用 2~6 字中文短语切分 + 去停用词（100+ 个虚词如"的""了""是""这个"）+ 按频率取 top 5。
