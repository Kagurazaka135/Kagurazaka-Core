# memory.py — 轻量级记忆系统

**让 AI 记得"刚才说了什么"。** 不是 RAG，不是向量数据库，就是最朴素的上下文追踪。

## 三层记忆结构

```
┌─ 工作记忆 (deque, 最大 7 条)
│  当前会话的关键信息, FIFO
│
├─ 会话历史 (deque, 最大 10 条)
│  最近 10 轮对话的完整记录
│
└─ 长期记忆 (磁盘 JSON)
   按 chat/code/logic 分类存储，每类上限 15~50 条
```

## 核心方法

### enhance_input(user_input) → str
**检测上下文引用，自动补上一个上文。**

```python
# 用户说 "那这个bug怎么修"
# 系统检测到"这个"指代上文，自动拼成:
"""
相关上下文:
[历史] 用户: 我的网页报500错误
助手: 可能是后端服务没启动...
当前问题: 那这个bug怎么修
"""
```

原理：正则匹配"刚才/刚刚/这个/那个/基于...的/继续"等指代词，或者检测话题关键词重叠。

### add_to_history(user_input, ai_output, task_type)
**每轮对话结束时调，保存进历史。** 超过 200 字的输出或非聊天类型的任务，额外保存到长期记忆。

### update_task_context(task_type, step_info)
**多步骤任务追踪。** 比如用户连续问代码问题，系统知道"上一个任务是代码，已完成 3 步"。

## 为什么不用向量数据库

这个系统的记忆需求很简单——就是"记得上一轮说了啥"和"这个话题是不是在延续"。对于这种场景，关键词提取 + 话题重叠检测比 embedding 检索更准、更快、更轻。

## 中文关键词提取

```python
_extract_keywords("我的Python爬虫老是超时怎么办")
# → {"Python", "爬虫", "超时", "怎么办"}
```

用 2~6 字的中文短语切分 + 去停用词（如"的""了""是""这个"等 100+ 个虚词）+ 按频率取 top 5。
