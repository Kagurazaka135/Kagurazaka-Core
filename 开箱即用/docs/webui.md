# webui.py — Gradio 网页前端

**给普通人用的聊天界面。** 244 行，`python webui.py` 启动。

## 交互流程

| 操作 | 触发的事件 | 发生了什么 |
|---|---|---|
| 切换供应商 | `on_provider_change()` | 7 个模型输入框自动填入该供应商的预设值 + base_url |
| 改模型名 | 手动编辑输入框 | 什么也不触发，只是编辑 |
| 点 💾 保存 | `save_settings()` | 收集所有输入 → 写 config.json → `init_config()` 热重载 |
| 点 🔍 测试 | `test_connection()` | 拿当前配置调一次分类器 LLM，显示成功/失败 |
| 输入按 Enter | `chat_fn()` | 调 `core.run_workflow()`，结果追加到聊天区 |
| 点 🗑 | `clear_chat()` | 清空聊天 + 新建记忆实例 |

## 关键细节

**记忆是会话级的**: 每个浏览器 Tab 有自己独立的 `SimpleMemorySystem`（通过 `gr.State()` 实现）。关掉 Tab 记忆丢失，刷新 Tab 记忆也重置。

**设置保存不需要重启**: `save_settings` 写 `config.json` 后调 `init_config()`，`CONFIG` 全局变量在内存中被刷新。下一个对话请求自动用新配置。

**API Key 是密码框**: Gradio 的 `type="password"`，页面上显示为 `••••••`。但注意——会传到服务器端，也存在于 `config.json` 文件里。不要把这个文件上传到 GitHub。

**头像占位**: 顶栏左侧目前是一个渐变色背景的 🌸 文字。替换成图片只需改 HTML：
```python
gr.HTML('<img src="/path/to/avatar.png" class="avatar-img">')
```

## 如何给局域网其他人用

```python
ui.launch(server_name="0.0.0.0", server_port=7860, share=False)
```

`server_name="0.0.0.0"` 让局域网其他设备也能访问。对方浏览器打开 `http://你的IP:7860`。

如果开启了 `share=True`，Gradio 会生成一个临时公网链接——但注意 API key 在服务器端，不会通过 share 链接泄露。
